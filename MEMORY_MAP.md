# Hermes Memory Map

## Purpose

Hermes should maintain a durable memory map for Olufunke so important family, business, content, file, and operating-context information can be found later.

## Recommended architecture

- Notion: living memory map and index.
- Google Drive: source archive and generated files.
- Google Docs: long-form notes, reports, and PDF-ready documents.
- Google Sheets: trackers, lists, budgets, customer records, and structured logs.
- Gmail: mailbox source for routed messages.

## Memory categories

- Family Admin
- Business Operations
- Customers and Follow-ups
- Finance and Records
- Content and Brand
- Business Ideas and Research
- App Projects
- Reports and PDFs
- Inbox Notes
- Decisions and Preferences

## Capture rule

Save durable memory when the user asks for it, when a task creates an artifact that should be recoverable, or when a decision changes future behavior.

## Retrieval rule

Before starting a task that depends on prior context, check the memory map first, then use Drive, Docs, Sheets, or Gmail as the source layer.

## Storage rule

Do not store secrets in the memory map. Store only labels, decisions, file links, summaries, and non-sensitive operating context.
